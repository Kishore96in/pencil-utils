For the template units, the parameter is the directory into which the test output should be written. The target directory should contain the appropriate `.conf` file, for which examples are given.

Copy the `.service` and `.timer` files into `~/.config/systemd/user/`.

# Invocation examples
## Run pc_auto-test once
```bash
systemctl --user start pencil_test@$(systemd-escape -- /home/kishore/public_html/pencil_tests/master).service
```

## Enable periodic runs of pc_auto-test
Edit `pencil_test.timer` to contain the path to the test output directory, e.g.
```
Unit=pencil_test@-home-kishore-public_html-pencil_tests-master\x2dtemplatecheck.service
```
where the part of the service name between `@` and `.` is generated by
```sh
systemd-escape -- /home/kishore/public_html/pencil_tests/master-templatecheck
#-home-kishore-public_html-pencil_tests-master\x2dtemplatecheck
```
Then, run
```bash
systemctl --user enable --now pencil_test.timer
```
